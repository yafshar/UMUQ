Detailed Description

template<typename DataType>
class umuq::kNearestNeighbor< DataType, umuq::DistanceTypes::MAHALANOBIS >

Finding K nearest neighbors in high dimensional spaces using Mahalanobis distance.

This class finds the K nearest neighbors in high dimensional spaces using Mahalanobis distance. The Mahalanobis distance is a measure of the distance between a point and a distribution. It is a multi-dimensional generalization of the idea of measuring how many standard deviations away the point is from the mean of the distribution.
It is essentially a stretched Euclidean distance.

The Mahalanobis distance between two vectors \( \vec{x} \) and \( \vec{y} \) with dimensionality of \( d, \) from the same distribution with the covariance matrix \( \Sigma \) of size \( d \times d \) is:

\( d(\vec{x}, \vec{y}) = \sqrt{(\vec{x} - \vec{y})^T \Sigma^{-1} (\vec{x} - \vec{y})}.\)

In this implamentation, we use the L2 distance and simply stretch the actual dataset itself before performing any evaluations. This is done to avoid calculating the full rating matrix, for Mahalanobis distance metric in K nearest neighbor search.

We do nearest neighbor search only on the transformed space, where we stretch the actual dataset itself before performing any evaluations.

\( \begin{align} \nonumber \text{Square of the Mahalanobis distance} &= \\ \nonumber d(\vec{x}, \vec{y})^2 &= (\vec{x} - \vec{y})^T \Sigma^{-1} \underbrace{(\vec{x} - \vec{y})}_{\vec{X}} \\ \nonumber d(\vec{x}, \vec{y})^2 &= \vec{X}^T\Sigma^{-1}\vec{X} \\ \nonumber \text{by decomposing}~\Sigma^{-1}=LL^T \text{(Cholesky decomposition)} \\ \nonumber &= \vec{X}^T LL^T\vec{X} \\ \nonumber &={(L^T\vec{X})}^T{(L^T\vec{X})} \end{align} \)

This can be seen as the nearest neighbor search on the \( L^T\vec{X} \) with using the squared Euclidean distance.

Warning
Avoid doing the Cholesky decomposition at first, and inversion later. Since, it would result to numerical inaccuracy.
Doing this some of the neighbors are computed wrongly.
\( \begin{align} \nonumber d(\vec{x}, \vec{y})^2 &= \vec{X}^T\Sigma^{-1}\vec{X} \\ \nonumber \text{by decomposing}~\Sigma=LL^T \text{(Cholesky decomposition)} \\ \nonumber &= \vec{X}^T (LL^T)^{-1}\vec{X} \\ \nonumber &= \vec{X}^T L^{-T}L^{-1}\vec{X} \\ \nonumber &={(L^{-1}\vec{X})}^T{(L^{-1}\vec{X})} \end{align} \)

The covariance matrix \( \Sigma \) associated with Mahalanobis distance.

To avoid calculating the full rating matrix, for Mahalanobis distance metric in K nearest neighbor search, we will do nearest neighbor search only on the transformed space, where we stretch the actual dataset itself before performing any evaluations.

\( \begin{align} \nonumber \text{Square of the Mahalanobis distance} &= \\ \nonumber d(\vec{x}, \vec{y})^2 &= (\vec{x} - \vec{y})^T \Sigma^{-1} \underbrace{(\vec{x} - \vec{y})}_{\vec{X}} \\ \nonumber d(\vec{x}, \vec{y})^2 &= \vec{X}^T\Sigma^{-1}\vec{X} \\ \nonumber \text{by decomposing}~\Sigma^{-1}=LL^T \text{(Cholesky decomposition)} \\ \nonumber &= \vec{X}^T LL^T\vec{X} \\ \nonumber &={(L^T\vec{X})}^T{(L^T\vec{X})} \end{align} \)

This can be seen as the nearest neighbor search on the \( L^T\vec{X} \) with using the squared Euclidean distance.
So, we'll decompose \( \Sigma^{-1} = LL^T \) (the Cholesky decomposition), and then multiply sampling space points and query points by \( L^T \). Then we can perform nearest neighbor search using the squared Euclidean distance.
