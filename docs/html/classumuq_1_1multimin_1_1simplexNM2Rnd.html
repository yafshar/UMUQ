Detailed Description

template<typename DataType>
class umuq::multimin::simplexNM2Rnd< DataType >

The Simplex method of Nelder and Mead, also known as the polytope search algorithm. It uses a randomly-oriented set of basis vectors instead of the fixed coordinate axes around the starting point x to initialize the simplex.

Template Parameters
DataTypeData type

Reference:
Nelder, J.A., Mead, R., Computer Journal 7 (1965) pp. 308-313.

This implementation uses \( n+1 \) corner points in the simplex.

◆ init()

template<typename DataType >
bool umuq::multimin::simplexNM2Rnd< DataType >::init ( )
virtual

Initialize the minimizer.

It uses a randomly-oriented set of basis vectors instead of the fixed coordinate axes around the starting point x to initialize the simplex.
The final dimensions of the simplex are scaled along the coordinate axes by the vector step_size.
The randomization uses a simple deterministic generator so that repeated calls to functionMinimizer set for a given solver object will vary the orientation in a well-defined way.

Reference:
https://www.gnu.org/software/gsl/doc/html/multimin.html
